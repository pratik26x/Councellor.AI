🔗 Relationships & Constraints

👤
USER → AUTH_TOKEN 1 : N One user can have many active session tokens (multi-device login). Token is deleted on logout or expiry.
📋
USER → STUDENT 1 : 1 A registered user can fill one student profile form. Linked by email (not a hard FK, by design).
🏅
SCHOLARSHIP (standalone) 0 : N Scholarships are independent listings. Any user can browse and apply. No FK to User.
📖
RESOURCE (standalone) 0 : N Resources are independent. Download count tracked per resource. No FK to User.
🏠
ACCOMMODATION (standalone) 0 : N PG/Flat listings are independent. Any user can browse or submit a listing.
🍱
MESS_TIFFIN (standalone) 0 : N Mess/Tiffin services are independent listings. Any user can browse or submit.
PK = Primary Key
FK = Foreign Key
UQ = Unique
NN = Not Null
